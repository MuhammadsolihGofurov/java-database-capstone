# Smart Clinic Management System - User Stories

This document defines the core functionalities of the Smart Clinic Management System from the perspective of different user roles.

## 1. Patient User Stories
* **Registration & Login:** As a patient, I want to create an account and log in securely so that I can manage my medical appointments.
* **Search Doctors:** As a patient, I want to search for doctors by their name or specialty to find the right healthcare provider.
* **Book Appointments:** As a patient, I want to view a doctor's available time slots and book an appointment at my convenience.
* **View History:** As a patient, I want to view my past and upcoming appointments to keep track of my healthcare visits.

## 2. Doctor User Stories
* **Manage Availability:** As a doctor, I want to set and update my available working hours so that patients can see when I am free.
* **View Appointments:** As a doctor, I want to see a list of all patients who have booked appointments with me for a specific date.
* **Issue Prescriptions:** As a doctor, I want to create and save digital prescriptions for my patients during their visits.
* **Patient Records:** As a doctor, I want to search for patient details by email or phone number to review their medical background.

## 3. Admin User Stories
* **Manage Staff:** As an admin, I want to add or remove doctors from the system to keep the clinic's staff directory up to date.
* **System Oversight:** As an admin, I want to monitor all appointments and clinic activities to ensure smooth operations.
* **Generate Reports:** As an admin, I want to generate daily and monthly reports on appointments and doctor performance for clinic management.
* **Security:** As an admin, I want to ensure that only authorized users (Doctors and Patients) have access to their respective portals.
